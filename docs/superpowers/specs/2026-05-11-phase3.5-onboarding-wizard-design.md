# Phase 3.5: Onboarding Wizard — Design Spec

## Context

Phases 0-3 built foundation + tRPC + Brand Brief + Anti-AI Rules. Onboarding is the activation hook — captures brand voice on first run so every subsequent feature (ideas, drafts, scoring) has context. Pulled forward from Phase 8 because brand setup is prerequisite for all content features.

Clerk is installed (`@clerk/nextjs` v7.3.3) but NOT wired — ClerkProvider missing from root layout. No onboarding state tracking in schema. This phase fixes both.

## Decisions Made

- **Steps:** 4 custom steps (after Clerk handles auth). Welcome overview + 4 steps + completion.
- **Corpus capture:** 3 methods + templates (text paste, guided prompts, templates, JSON import). LinkedIn pull / file upload deferred to Phase 4A+.
- **Guardrails:** Category-level enable/disable with preset templates. Individual rules created lazily from templates when user visits /rules.
- **Multi-brand:** Primary brand in wizard. Additional brands from /brand page.
- **Skip policy:** Step 1 mandatory. Steps 2-4 skippable with warnings.
- **Knowledge base / file uploads:** Tracked for Phase 4+. Placeholder UI shown.

## Competitor Inspirations

- **Jasper AI:** Tone quick-selector (Professional/Casual/Witty) → adopted as voice style dropdown
- **Lately:** "Paste 5+ posts" with progress bar → adopted as corpus counter
- **Copy.ai:** Website URL auto-scrape → deferred to Phase 5, stored URL now
- **General:** "What to expect" welcome screen → reduces onboarding anxiety

## Flow

```
Clerk Auth (sign up / sign in)
  ↓
Auto-create workspace if not exists
  ↓
Redirect to /onboarding if onboardingStep < 5
  ↓
Welcome Overview (what to expect, 5 min total)
  ↓
Step 1: Brand Identity (30s, REQUIRED)
  ↓
Step 2: Voice Corpus (2-3 min, skippable)
  ↓
Step 3: Review Brand Brief (1-2 min, skippable)
  ↓
Step 4: Content Guardrails (30s, skippable)
  ↓
Complete → set onboardingStep=5 → redirect to dashboard
```

## Step Details

### Welcome Overview

- Eyebrow: "FIRST-RUN ACTIVATION"
- Title: "Welcome to Content Intelligence Agent"
- 4-step preview cards showing: Brand Identity → Voice Corpus → Brand Brief → Guardrails
- Total time estimate: "~5 minutes"
- CTA: "Get started"

### Step 1: Brand Identity (REQUIRED)

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| Company/brand name | text | Yes | Creates brand record |
| Industry | dropdown | Yes | SaaS, Agency, E-commerce, Fintech, Healthcare, Education, Other |
| Voice style | radio cards | Yes | Direct Operator / Storyteller / Data-Driven / Casual Expert / Authoritative |
| Your role | dropdown | No | Founder, Marketing Lead, Content Manager, Agency, Other |
| Single or multi-brand | radio | Yes | If multi: show "Add brand" inputs for additional brand names |
| Website URL | text | No | Stored for future auto-scrape (Phase 5). Not scraped now. |

**Backend:** `brand.create` (one per brand name). Stores voice style in workspace metadata or brand config.

### Step 2: Voice Corpus (SKIPPABLE)

**Progress indicator:** "X of 5 recommended samples" with colored bar (red <3, yellow 3-5, green 5+)

**Skip warning:** "Voice fidelity scoring stays disabled until you add at least 5 samples. We'll remind you after your first 5 posts."

**Three tabs:**

**Tab A: Paste your posts**
- Large textarea (min-height 200px)
- Helper: "Paste 10-20 of your best posts, separated by blank lines"
- Live counter: "X posts detected"
- "Add to corpus" button → splits on double newline → calls `corpus.add` per item

**Tab B: Guided voice prompts**
- 5 structured prompts, each with textarea:
  1. "Paste your single best-performing LinkedIn/social post"
  2. "Explain your product/service like you'd tell a friend at dinner"
  3. "What's a contrarian opinion you hold about your industry?"
  4. "Write the opening paragraph of your ideal newsletter"
  5. "List phrases you ALWAYS use, and phrases you NEVER use"
- "Save all answers" → calls `corpus.add` per non-empty answer

**Tab C: Start from a template**
- 4 voice style cards (matching Step 1 selection highlighted):
  - **Direct Operator** — "Short paragraphs. Specific numbers. Contrarian assertions. Zero fluff."
  - **Storyteller** — "Narrative arcs. Vulnerability. 'Here's what happened when...' structure."
  - **Data-Driven Analyst** — "Benchmarks and charts. 'The data shows...' framing. Evidence-first."
  - **Casual Expert** — "Conversational. Contractions. Questions as hooks. First-person."
- Each template has 5 sample posts user can edit
- "Use this template" → pre-fills corpus with 5 editable posts

**Optional: Import JSON**
- Collapsed section: "Have a Salience export or JSON file?"
- Textarea for pasting JSON
- Parse and insert as corpus items

**Knowledge base placeholder:**
- Grayed-out section below tabs
- "Upload brand guidelines, case studies, or reference docs"
- Badge: "Coming in next update"
- Stored for Phase 4+

**Backend:** `corpus.add` (per item). Items stored without embeddings (Phase 5).

### Step 3: Review Brand Brief (SKIPPABLE)

**If corpus ≥ 5 items:** Brief fields pre-populated from voice style selection + corpus analysis placeholder text:
- Wedge: auto-generated from voice style (template-based, not LLM)
- ICP: blank (user fills)
- Voice Traits: from template matching selected voice style
- Anti-Positioning: blank (user fills)

**If corpus < 5 items:** All fields blank, user fills manually.

**All fields editable regardless.**

| Field | Pre-populated? | Required |
|-------|---------------|----------|
| Wedge | Template-based | No (skippable step) |
| ICP | No | No |
| Voice Traits | Template-based | No |
| Anti-Positioning | No | No |

**Advanced section (collapsed):**
- Preferred channels: multi-select (LinkedIn, X, YouTube, Instagram, Newsletter, Blog, Reddit, Other)
- Content purpose: dropdown (Thought leadership, Lead generation, Community building, Brand awareness)

**Backend:** `brief.create` (version 1). Skipped = no brief created (brand.get returns `brief: null`).

### Step 4: Content Guardrails (SKIPPABLE)

**Category-level toggles** (not 62 individual rules):

| Category | Default | Preset Count |
|----------|---------|-------------|
| Punctuation | ON | 8 rules |
| Transition | ON | 12 rules |
| Filler | ON | 10 rules |
| Corporate | ON | 15 rules |
| Cliché | ON | 12 rules |
| Custom | OFF | 5 rules |

**Strict mode toggle:** ON by default. Description: "Drafts with violations cannot be published."

**Skip behavior:** If skipped, all categories ON + strict mode ON (safe defaults).

**Backend:** Store enabled categories as workspace-level config. Actual 62 rules inserted lazily when user first visits /rules page (from rule templates). Avoids 62 inserts during onboarding.

**Rule templates:** Static JSON file `src/lib/rules/default-rules.ts` with 62 pre-defined rules. Inserted on first /rules visit if no rules exist for workspace.

## tRPC Procedures

| Router | Procedure | Type | Purpose |
|--------|-----------|------|---------|
| onboarding | getStep | query | Return current onboarding step for workspace |
| onboarding | saveBrandIdentity | mutation | Step 1: create brand(s), save industry/voiceStyle |
| onboarding | saveCorpusItems | mutation | Step 2: batch insert corpus items |
| onboarding | saveBrief | mutation | Step 3: create brief v1 (calls brief.create) |
| onboarding | saveGuardrails | mutation | Step 4: save category preferences + strict mode |
| onboarding | complete | mutation | Set step=5, completedAt=now() |
| onboarding | skip | mutation | Advance step without saving (set step to next) |

## Edge Cases

| Case | Handling |
|------|---------|
| User without Clerk org | Auto-create personal workspace + org on first sign-in |
| Browser close mid-wizard | `onboardingStep` persisted. Resume on return. |
| Multi-brand selected | Primary brand through wizard. Others created as records, setup from /brand. |
| Corpus skip | Voice scoring disabled. Reminder banner on dashboard + after 5 posts. |
| Brief skip | brand.get returns brief: null. Prompt to create from /brand page. |
| Back navigation (Step 3→2) | Data persisted per step. No regeneration needed. |
| Existing workspace (already onboarded) | Skip redirect. Direct to dashboard. |
